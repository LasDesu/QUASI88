
		QUASI88 のコンパイル方法  for Win32 + Visual C++ 6.0

						2006/09/22
						福永省三
						e-mail : fukuchan@mh1.117.ne.jp
-------------------------------------------------------------------------------

●ここでは Windows 環境の人向けに、QUASI88 を VC++ 6.0 を使ってコンパイルする
  方法を説明します。


	1) QUASI88のソースを、展開 (unzip) してください。
	   ソースファイルの入っている、 src というフォルダおよび、
	   VC++ 6.0用のプロジェクトファイル QUASI88.dsp 、 QUASI88.dsw
	   などが展開されます。


	2) SDLのホームページ http://www.libsdl.org/ より、
	   SDLの開発者向けライブラリ (Development Libraries) の、
	   Win32 - Visual C++ 5,6,7 版を入手します。

	   2006/09/10現在、SDLのバージョンは 1.2.11 です。URLは以下のとおり。

		http://www.libsdl.org/release/SDL-devel-1.2.11-VC6.zip

	   これを展開 (unzip) すると、 SDL-1.2.11 というような名前のフォルダが
	   できるので、このフォルダの名前を、 SDL-devel に変更してください。
		       ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
	   変更したら、このフォルダをまるごと  1) のフォルダ ( srcフォルダや、
	   QUASI88.dsw などのファイルのあるフォルダ ) にコピーします。


	3) QUASI88.dsw をダブルクリックすると、VC++ が起動するので、
	   ビルドしてください。

	   プロジェクトの設定には、「Release」「Monitor」「Debug」が
	   あります。これらの違いは以下の通りです。

		Release … ウインドウアプリケーションとして実行ファイルが生成
			   されます。QUASI88のモニタモードは使用できません。

		Monitor … コンソールアプリケーションとして実行ファイルが生成
			   されます。QUASI88のモニタモードが使用可能です。

		Debug   … コンソールアプリケーションとして実行ファイルが生成
			   されます。QUASI88のモニタモードが使用可能です。
			   実行ファイルはデバッグ情報付きで、最適化されずに
			   コンパイルされています。
			   デバッグ用なので、配布しないでください。

	   （このプロジェクトファイルにて生成される実行ファイルには、
	     MAME/XMAMEベースのサウンド出力 と fmgen が組み込まれます。）


	4) ビルドが完了したらコントロールキーを押しながら、F5 キーを押すと 
	   QUASI88.exe が起動します。この時、「ダイナミック リンク ライブラリ 
	   SDL.dll が指定されたパスに見つかりませんでした。」と表示される場合
	   は、SDL.dll を 1) のフォルダにコピーしておいてください。
	   ( カレントディレクトリか、パスの通ったディレクトリにあれば問題ない
	     はずです )

	   なお、 SDL.dll は、 2) で展開した SDLの開発者向けライブラリの、
	   lib フォルダにもあるはずです。
	   SDLのホームページ http://www.libsdl.org/ からも入手可能です。


	5) ここからは、お好みで好きなように設定を変更してみてください。


	*) QUASI88 の改変・再配布などに関しては、もともとの QUASI88 の
	   著作権・免責事項に加えて以下を守ってください。

		QUASI88 の実行には、 SDL.dll が必要です。
		SDL.dll をセットにして配布する場合は、 SDL に付属の
		README-SDL.txt をいっしょに配布してください。
		(SDLのドキュメントに、そう書いてあるので・・・)
		SDL のライセンスにも従ってください。

		MAME/XMAMEベースのサウンド出力を組み込んだ場合、
		MAME/XMAME のライセンスに従う必要があります。

		fmgenを組み込んだ場合、
		fmgen のライセンスに従う必要があります。

		実行ファイルと同じフォルダに、以下の空のフォルダを作成して
		おくとベターです。これらのフォルダは、設定のない状態で 
		QUASI88 を実行した時の各種参照フォルダになります。

			INI   フォルダ
			ROM   フォルダ
			DISK  フォルダ
			TAPE  フォルダ
			SNAP  フォルダ
			STATE フォルダ
